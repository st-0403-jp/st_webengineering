ディレクトリ・ファイル名

  • 全て小文字で a-z 0-9 . - _ を使用可能な文字とする。
  • 連番や日付はソートや拡張性を考慮する

パスの記述方法

  • index.htmlを指定する場合は、ファイル名を省略して / で終わること。
<a href="index.html">トップ</a><!-- NG -->
<a href="/">トップ</a><!-- OK -->
  • // から始まるプロトコルレラティブURIを推奨する。
<script src="http://x.gnst.jp/s.js"></script> <!-- 非推奨 -->
<script src="//x.gnst.jp/s.js"></script> <!-- 推奨 -->

キャッシュ対策

  • CSS/JS/画像ファイルを変更した場合はURLにパラメータを追加してキャッシュ対策をする。
  • パラメータにはタイムスタンプもしくはリビジョンなどユニークな値を指定する。
<link rel="stylesheet" href="css/style.css?201500901"> <!-- タイムスタンプ -->
<link rel="stylesheet" href="css/style.css?44011"> <!-- リビジョン -->

アクセス解析（google analytics）

下記トラッキングコード（計測タグ）をbody要素末尾に埋め込むこと。

<div style="display:none;height:0;position:relative;visibility:hidden;width:0;">
<script src="//x.gnst.jp/s.js"></script>
<script>('localhost' !== location.hostname) && document.write(unescape("%3Cscript src='//site.gnavi.co.jp/analysis/sc_"+getScSubdom()+".js'%3E%3C/script%3E"));</script>
</div>
</body>
</html>

コピーライト（copyright）

「ぐるなび著作権表示（コピーライト）について ＞ 著作権表示規定」に則って対応すること。

<p><small>&copy; Gurunavi, Inc.</small></p>

HTML5

  • html5shiv.js はHTMLの要素が変更されるので使用しない。
  • 終了タグを省略しない
  • 空要素の終端スラッシュは不要（記述しない）
  • 要素名と属性名は必ず小文字にする
  • 属性は原則として属性名と属性値の双方を記述する
  • 属性値は原則として"で囲むこと

検証ツール

仕様準拠の確認はHTML5に対応したバリデータを利用する。

  • http://validator.w3.org
  • http://html5.validator.nu/

ただし、ソーシャルモジュールを使っている場合に限り、以下の警告は例外として許容する。

  • iframe[frameborder]
  • iframe[scrolling]

SPページの場合は、ページの設計がモバイルフレンドリーであるか確認する。

  • モバイルフレンドリー テスト

Webアクセシビリティの考慮

  • 人が見てわかりやすい（ヒューマンリーダブルが高い）
  • 機械が読みやすい（マシンリーダブルが高い）
Webアクセシビリティ
├ ヒューマンリーダブル
    - 情報の認識が容易
    - 代替情報の最適化
└ マシンリーダブル
    - Web標準
    - 的確なmeta情報
    - 文書構造（アウトライン）
    - セマンティックマークアップ
    - WAI-ARIA
    - 構造化データ

focusの可視化

サイトのキーボード操作が出来なくなるため、outlineを単純に非表示にする処理を禁止する。

  • 推奨処理は下記の通り
    • デフォルトのまま
    • 非表示処理をする場合はデザインによる装飾を指定し、フォーカスがどこにあるのか常時確認可能な状態であること
/* NG */
* {
  outline: none;
}
*:focus {
  outline: 0;
}

DOCTYPE宣言　

  • 必須
  • 先頭に必ず配置する
<!DOCTYPE html>

html[lang="**"]

  • 音声合成ツールの言語選択、および翻訳ツールの規則決定のため指定を推奨する
  • 日本語以外の文書は、別途lang属性を最適化すること
<html lang="ja">

title

  • 必須
  • SEOガイドラインを参照　→　SEO Guideline > title

meta[name="viewport"]

  • SP 推奨
  • 特に指定がない場合は以下を指定する
    • width=device-width
    • user-scalable=no
    • initial-scale=1
    • maximum-scale=1
<meta name="viewport" content="width=device-width,user-scalable=no,initial-scale=1,maximum-scale=1">

meta[name="format-detection"]

  • SP 推奨
  • 特に指定がない場合は以下を指定する
    • telephone=no
<meta name="format-detection" content="telephone=no">

meta[charset="**"]

  • 要素先頭で必ず配置する
<meta charset="utf-8">

meta[name="description"]

  • 推奨
  • SEOガイドラインを参照　→　SEO Guideline > meta-description

meta[name="keywords"]

  • 推奨
  • SEOガイドラインを参照　→　SEO Guideline > meta-keywords

meta[http-equiv="X-UA-Compatible"]

  • PC 推奨
  • インストールされているIEの最新バージョンの標準モード表示になるよう content="IE=edge" を設定する
  • CSSやJSなどの外部ファイルの読み込みよりも前に記述する

link[rel="canonical"]

  • 推奨
  • 重複URIの正規化が必要であれば記述する
  • 絶対パスで記述する
  • 重複URIの正規化が必要なケース
    • URLの表記に一貫性が保たれてない場合
    • URLにパラメータが付与され、動的にURLが複数生成される場合
    • PCとSP・FPで異なるURI構造をもつ場合（SEO Guideline > link-alternate ）

link[rel="alternate" hreflang="x"]

  • 任意
  • グローバル対応している場合記述する
  • ページが閲覧対象とする国・地域・言語を追加する
  • link[rel="canonical"]と組み合わせることで、個々のウェブページがどの国・言語のユーザーに閲覧されることを想定しているかということを検索エンジンへ伝えることが可能
<link rel="alternate" href="https://gurunavi.com/" hreflang="en">
<link rel="alternate" href="https://gurunavi.com/zh-hans/" hreflang="zh-Hans">
<link rel="alternate" href="https://gurunavi.com/zh-hant/" hreflang="zh-Hant">
<link rel="alternate" href="https://gurunavi.com/ko/" hreflang="ko">
<link rel="alternate" href="https://gurunavi.com/th/" hreflang="th">
<link rel="alternate" href="https://gurunavi.com/id/" hreflang="id">
<link rel="alternate" href="https://gurunavi.com/ms/" hreflang="ms">

link[rel="shortcut icon"]

  • 非推奨
  • HTMLによる指定は明示せずに、各サービスのドキュメントルートに ぐるなび標準のファビコン を設置する
  • 各サービスでオリジナルのファビコンを用意する場合でも、ファビコン名は favicon.ico とする

link[rel="apple-touch-icon"]

  • SP 推奨
  • オリジナルのアイコンに角丸の効果を加えて表示される
  • Webサイトへのショートカットをホーム画面に追加した際に作られるブックマークアイコンになる
<link rel="apple-touch-icon" href="//x.gnst.jp/webclip.png">

link[rel="stylesheet"]

  • CSSガイドラインを参照 → CSS Guideline　ファイル数、読み込み位置

コンディショナルコメント

  • 非推奨
<!--[if IE 7]>
<link rel="stylesheet" href="ie7.css">
<![endif]-->

<!--[if lte IE 6]>
<link rel="stylesheet" href="ie6.css">
<![endif]-->

script

  • javascriptガイドラインを参照　→　JavaScript Guideline　ファイル数、<script>タグの記述場所

iframe

  • 非推奨
  • ソーシャルモジュール以外では非推奨
  • ターゲットブラウザでiframeの表示スタイルを保持するために下記の属性は必須で設定する
    • iframe[frameborder]
    • iframe[scrolling]

h1

  • 必須
  • 必ず1つ配置する
  • ターゲットキーワードを必ず含むこと
  • h1はbodyに対しての見出しなのでセクショニングコンテンツ（section, article, nav, aside)で囲まない
  • SEOガイドラインを参照　→　SEOガイドライン > hx

h2, h3, h4, h5...

  • セクショニングコンテンツとペアで利用すること
  • 見出しレベルはページの上から順になること

alt属性

  • SEOガイドラインを参照　→　SEOガイドライン > alt

title属性

  • SEOガイドラインを参照　→　SEOガイドライン > title

role属性

  • 推奨

  • role=banner

    • header要素に追加するロール
    • ロゴ/名前やサイト固有のその他の情報が含まれる
    • ドキュメントで1回だけ使用できる

  • role="complementary"

    • aside要素に追加するロール
    • メインコンテンツと多少関係があるコンテンツを示す

  • role="navigation"

    • nav要素に追加するロール

  • role="search"

    • 検索用form要素に追加するロール

  • role="contentinfo"

    • 対象はfooter要素に限らない
    • 著作権やプライバシーポリシー及びそれらのリンクを示す
    • ドキュメントで1回だけ使用できる
<header role="banner">
<aside role="complementary">
<nav role="navigation">
<form role="search">
<footer role="contentinfo">
